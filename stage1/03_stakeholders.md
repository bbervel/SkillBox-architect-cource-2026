# 3. Анализ стейкхолдеров и их интересов

| Стейкхолдер               | Роль / Описание                                                                                         | Интересы и ожидания                                                                                                                                                                                                                                                                                              | Влияние на архитектуру                                                                                                                                                                                                                                                                                        |
|---------------------------|---------------------------------------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Пользователи**          | Конечные потребители: любители, профессионалы, новички, дети, пожилые, путешественники.                | • Удобный интерфейс и простота использования.<br>• Мотивация и геймификация.<br>• Социальное взаимодействие (группы, общение).<br>• Точное отслеживание тренировок и прогресса.<br>• Конфиденциальность и контроль над данными.<br>• Персонализированные рекомендации и промо.                                   | • Требования к производительности (быстрый отклик).<br>• Необходимость поддержки разных языков, локалей.<br>• Безопасное хранение данных и настройки приватности.<br>• Поддержка различных устройств (мобильные платформы, носимые устройства).<br>• Проектирование интерфейсов, ориентированных на пользователя. |
| **Маркетинговый отдел**   | Отвечает за продвижение бренда, привлечение пользователей, проведение промоакций.                      | • Возможность запускать таргетированные промоакции на основе данных тренировок.<br>• Инструменты для сбора аналитики о поведении пользователей.<br>• Интеграция с CRM и системами лояльности.<br>• Увеличение узнаваемости бренда через социальные функции.<br>• Поддержка региональных маркетинговых кампаний. | • API для управления промоакциями и их таргетинга.<br>• Сбор и хранение событий для аналитики (event logging).<br>• Возможность A/B тестирования.<br>• Гибкая система управления контентом (CMS) для промо-материалов.                                                                                       |
| **Продуктовые менеджеры** | Определяют функциональность, приоритеты развития, следят за метриками продукта.                        | • Гибкость в добавлении новых функций.<br>• Понятная аналитика использования (user engagement).<br>• Возможность быстро экспериментировать (feature toggles).<br>• Сбор обратной связи от пользователей.<br>• Соответствие рыночным трендам.                                                                    | • Модульная архитектура (микросервисы/плагины) для независимого развертывания фич.<br>• Наличие feature flags.<br>• Интеграция с системами аналитики (Google Analytics, Mixpanel).<br>• API для получения метрик.                                                                                         |
| **Разработчики**          | Команды, реализующие функциональность (backend, mobile, web, QA).                                      | • Понятная, документированная архитектура.<br>• Современный технологический стек.<br>• Возможность параллельной работы без конфликтов.<br>• Автоматизация тестирования и CI/CD.<br>• Легкость отладки и мониторинга.<br>• Хорошая поддержка сообщества и инструментов.                                          | • Выбор технологий, знакомых команде.<br>• Контейнеризация, оркестрация (Kubernetes).<br>• Единые стандарты кодирования и API.<br>• Инструменты для кодогенерации и документации (Swagger/OpenAPI).<br>• Системы логирования и трассировки.                                                                  |
| **DevOps / Инфраструктура** | Обеспечивают развертывание, масштабирование, надежность и безопасность инфраструктуры.                | • Масштабируемость под глобальную аудиторию.<br>• Отказоустойчивость и высокое время безотказной работы (SLA).<br>• Наблюдаемость (метрики, логи, трейсы).<br>• Управление затратами на облачные ресурсы.<br>• Автоматизация инфраструктуры (IaC).<br>• Безопасность сетей и доступа.                            | • Выбор облачных провайдеров с поддержкой регионов.<br>• Проектирование горизонтального масштабирования.<br>• Использование CDN, балансировщиков.<br>• Инструменты мониторинга (Prometheus, Grafana, ELK).<br>• Резервное копирование и disaster recovery.                                                 |
| **Региональные менеджеры** | Отвечают за развитие бизнеса в конкретных странах/регионах.                                           | • Возможность проводить локальные промоакции.<br>• Соответствие местным законам (GDPR, CCPA и др.).<br>• Локализация интерфейса и контента.<br>• Поддержка местных языков и культурных особенностей.<br>• Интеграция с местными платежными системами (если потребуется).                                       | • Архитектура, поддерживающая мультиязычность и локализацию (i18n).<br>• Конфигурируемые правила для промо (региональные фильтры).<br>• Хранение данных с учетом региональных требований (data residency).<br>• Гибкие настройки приватности.                                                               |
| **Партнёры**              | Производители устройств (Garmin, Polar, Apple), спортивные клубы, тренеры, медиа-компании.            | • API для интеграции с устройствами и обмена данными (с согласия пользователей).<br>• Возможность размещать свои тренировки, программы, контент.<br>• Доступ к обезличенной статистике для исследований.<br>• Совместные маркетинговые активности.                                                               | • Разработка открытых API с OAuth2 аутентификацией.<br>• Поддержка стандартных протоколов (BLE, ANT+, HealthKit, Google Fit).<br>• Механизмы управления согласиями пользователей на передачу данных партнёрам.<br>• Sandbox-среды для тестирования интеграций.                                             |
| **Юридический отдел / Compliance** | Обеспечивают соблюдение законодательства, защиту данных.                                              | • Соответствие GDPR, CCPA и другим регуляциям.<br>• Четкая политика конфиденциальности и условия использования.<br>• Контроль за сбором, хранением и удалением данных.<br>• Защита интеллектуальной собственности.<br>• Механизмы получения согласий.                                                            | • Встроенные механизмы управления согласиями (consent management).<br>• Возможность экспорта и удаления данных пользователя (right to be forgotten).<br>• Шифрование данных в покое и при передаче.<br>• Аудит доступа к данным.                                                                           |
| **Топ-менеджмент / Инвесторы** | Принимают стратегические решения, выделяют бюджет.                                                    | • Рост пользовательской базы и вовлеченности (DAU/MAU).<br>• Увеличение продаж через приложение.<br>• Окупаемость инвестиций (ROI).<br>• Конкурентное преимущество.<br>• Положительный имидж бренда.                                                                                                            | • Архитектура должна позволять быстро выводить продукт на рынок (time-to-market).<br>• Масштабируемость для поддержки роста.<br>• Низкая стоимость владения (TCO).<br>• Возможность интеграции с бизнес-аналитикой для отчетности.                                                                         |
| **Служба поддержки пользователей** | Помогают пользователям решать проблемы, отвечают на вопросы.                                         | • Доступ к данным о проблемах (логи, история действий).<br>• Инструменты для управления обращениями (ticketing).<br>• Возможность быстрого уведомления разработчиков о критических багах.<br>• База знаний и FAQ.                                                                                               | • Логирование действий пользователя (с согласия).<br>• API для интеграции с CRM поддержки.<br>• Возможность создания дашбордов с частыми проблемами.<br>• Механизмы обратной связи в приложении.                                                                                                           |
| **Финансовый отдел**      | Контролируют бюджеты, прогнозируют доходы и расходы.                                                  | • Прогнозируемые затраты на инфраструктуру.<br>• Отслеживание доходов от продаж через приложение.<br>• Прозрачность затрат на разработку.<br>• Интеграция с финансовыми системами (бухгалтерия).                                                                                                                | • Мониторинг затрат на облачные ресурсы (cost tagging).<br>• Механизмы биллинга (если будут платные функции).<br>• API для выгрузки финансовых данных.                                                                                                                                                     |
| **Аналитики данных**      | Строят отчеты, модели, исследуют поведение пользователей.                                             | • Доступ к «чистым» данным (без PII).<br>• Гибкие инструменты для запросов (SQL, BigQuery).<br>• Возможность строить дашборды.<br>• Интеграция с Data Warehouse компании.                                                                                                                                       | • Создание Data Lake / Data Warehouse.<br>• ETL/ELT процессы.<br>• Анонимизация данных.<br>• Поддержка экспорта данных в аналитические системы.                                                                                                                                                             |
| **Отдел продаж (e-commerce)** | Отвечают за интернет-магазин, увеличение продаж товаров компании.                                    | • Бесшовная интеграция с каталогом товаров.<br>• Персонализированные рекомендации товаров на основе тренировок.<br>• Отслеживание конверсии из приложения в покупку.<br>• Интеграция с корзиной и платежными системами.                                                                                          | • API для получения данных о товарах и ценах.<br>• Механизмы реферальных ссылок.<br>• Поддержка веб-вью или нативного магазина.<br>• События для аналитики продаж.                                                                                                                                        |
| **Команда QA**            | Обеспечивают качество продукта, тестируют функциональность.                                           | • Возможность автоматизации тестирования.<br>• Изолированные тестовые среды.<br>• Доступ к логам и трассировке.<br>• Инструменты для нагрузочного тестирования.<br>• Четкие критерии приемки.                                                                                                                  | • Наличие feature toggles для тестирования в production.<br>• Инфраструктура для тестовых окружений (staging).<br>• Архитектура, поддерживающая мокирование внешних сервисов.<br>• Логирование для отладки.                                                                                                |
